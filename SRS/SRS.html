1.Introduction

1.1 Purpose

We are aiming to build a Classroom which is super minimal so it's easy to learn, use, and share with other teachers, students, and families. It is crammed full of features and is always updating with new integrations that allow you to access lots of free-to-use tools aimed to help make teaching and learning better. While this isn't an LMS (Learning Management System), such as Blackboard, it complements many ideally and, in many cases, integrates as well. Yet as this Classroom becomes more powerful than ever, it is fast representing a new way to teach and manage the class online and remotely.

2. Overall Description

2.1 Product Perspective

One of the biggest benefits of Our Classroom is it’s simple to use and encourages collaboration between students and teachers. Teachers can create a class and list educational apps, such as assignments, in a few clicks. They can add students by sending them a code to join. Students can then see what assignments are due, participate in discussion forums or message the teacher (either in private or via group chat). Classroom integrates with many student information systems, along with other third-party applications, such as instructional tools, digital literacy solutions and coding software. And this management system will also track student activities and send them to teachers so that students always remain engaged with their studies.

3. External Interface Requirements

3.1 User Interfaces

It's mainly the comfortness of user when he/she using this application. and also the gui of this project how it styled. the loading, redirect link , button working writely . the webcam took pictures on right time , the availability of stored data all are presented on a efficient screen interface.


3.2 Hardware Interfaces

Since the application must run over the internet, all the hardware shall require to connect internet will be hardware interface for the system. As for e.g. Modem, WAN – LAN, Ethernet Cross-Cable.


3.3 Software Interfaces

  1. Teachers and students should be able to sign up to the system.
  2. Students need to provide a recent photograph when signing up.
  3. Teachers should be able to create classroom
  4. Teachers should be able to send invitation code to students
  5. Students can join classroom through invitation code
  6. Teacher can assign PDF files as reading material to students in a class
  7. Students can see their reading tasks and deadlines on a per-class basis but also as a unified list
  8. When a student decides to read a particular material, the system will open and display the material by itself (no download allowed)
  9. Material will be presented one page/slide/content-group at a time
  10. Students should be able to zoom in and out in the material
    10.1 Questions can be written down
    10.2 Questions can be asked verbally, which the system will record
    10.3 Questions can be tagged to particular parts of the material which the student can highlight
  11. Student questions are recorded/stored and are accessible to the teacher through a suitable interface. This interface should also present any relevant section the student has highlighted while asking the question.
  12. When students are reading the material, the system will take photographs of the student via the webcam or the phone camera. These photographs will be stored in the back end and the teacher can see a reel of these images tagged with the content being displayed.
  13. The system should run a face recognition module to ensure that the person in front of the camera is indeed the student and not someone else.

3.4 Communications Interfaces

This project supports all types of web browsers and android devices and the database is same. The system shall use the HTTP protocol for communication over the internet and for the intranet communication will be through TCP/IP protocol suite

5. Other Nonfunctional Requirements

5.1 Performance Requirements

The system will be interactive, and the delays involved will be less. So, in every action-response of the system, there are no immediate delays. In case of opening windows, forms of popping error messages and saving the settings or sessions there is delay much below 2 seconds, in case of opening databases, sorting questions and evaluation there are no delays, and the operation is performed in less than 2 seconds for opening, sorting, computing, posting the files. The performance of the system is dependent on how the load is delivered to the system. Due to increase workload, a peak workload will be defined.


5.2 Safety Requirements

System failure, login issues, account creation and password reset failure will cause damage or loss to the system. A backup system will be there for system failure which will be in archival storage.Information transmission should be securely transmitted to server without any changes in information.


5.3 Security Requirements

Security requirements includes authentication, authorization, data protection and so on . Authentication ensures that each entity involved in using a the service—the requestor, the provider, and the broker —is what it actually claims to be. Authorization determines whether the service provider has granted access to the Web service to the requestor. Data protection ensures that the Web servicerequest and response have not been tampered with en route. The main security concern is for users account hence proper login mechanism should be used to avoid hacking. The tablet id registration is way to spam check for increasing the security. Hence, security is provided from unwanted use of recognition software.
